## 1. 基础使用场景

### 1.1 快速开始：一篇文档 → 一个 Wiki

```bash
# 创建项目
mkdir my-wiki && cd my-wiki

# 一键摄入+编译+打开查看器
llmwiki quickstart https://en.wikipedia.org/wiki/Border_Gateway_Protocol

# 结果：wiki/concepts/ 下有编译好的互连页面
#      wiki/index.md 是自动生成的目录
#      自动打开浏览器查看器
```

### 1.2 知识问答

```bash
# 基于编译后的 Wiki 回答问题
llmwiki query "What are the BGP path attributes?"

# 将答案固化为 Wiki 页面（知识积累）
llmwiki query "What is the difference between eBGP and iBGP?" --save

# 调试模式：查看检索到的页面和 chunk
llmwiki query "How does route reflector work?" --debug
```

### 1.3 知识审查工作流

```bash
# 方式一：编译即审查（--review 标志）
llmwiki compile --review
# 所有生成页面暂存到 .llmwiki/candidates/

# 查看审查队列
llmwiki review list

# 查看具体候选页
llmwiki review show <candidate-id>

# 批准
llmwiki review approve <candidate-id>

# 拒绝（归档到 .llmwiki/candidates/archive/）
llmwiki review reject <candidate-id>
```

### 1.4 知识修复

```bash
# 查看哪些页面过时或成为孤儿
llmwiki status

# 预览修复计划
llmwiki refresh --stale --dry-run

# 执行修复
llmwiki refresh --stale
```

### 1.5 知识导出

```bash
# 导出为 OKF（Open Knowledge Format）
llmwiki export --target okf --out ./dist/okf

# 导出为 JSON
llmwiki export --target json

# 导出为 GraphML（图可视化）
llmwiki export --target graphml

# 导出为 Marp 幻灯片
llmwiki export --target marp

# 导出为 llms.txt（供其他 LLM 工具消费）
llmwiki export --target llms.txt
```

### 1.6 CI 质量门禁

```bash
# 运行 lint（检查断链、孤儿页、空页、断引用）
llmwiki lint

# 快速评估（无需 LLM）
llmwiki eval --suite fast

# 完整评估（含 LLM 评判引文支持度）
llmwiki eval --suite full --record
```

## 2. 高级使用场景

### 2.1 大规模离线编译

```bash
# 批量化摄入
llmwiki ingest https://example.com/doc1.md
llmwiki ingest https://example.com/doc2.md
llmwiki ingest /path/to/local/doc3.md

# 或者批量摄入会话导出
llmwiki ingest-session ~/claude-sessions/session1.json
llmwiki ingest-session ~/codex-sessions/session2.json

# 高并发编译
llmwiki compile --concurrency 20

# 指定输出语言
llmwiki compile --lang zh-CN
```

**批量摄入最佳实践：**
1. 分批编译（每批 20-30 个源文件）
2. 每批之间运行 `llmwiki lint` 检查质量
3. 使用 `--concurrency` 根据 provider 配额调节
4. 大文件拆分为小文件（避免触发 100K 字符截断）

### 2.2 领域建模：为你的场景定制 CLP

这是 v1.0 最强大的能力。创建一个 `.llmwiki/profile.json` 来定义你领域专属的知识结构。

#### 核心结构：

```json
{
  "version": 1,
  "id": "my-domain",
  "name": "My Domain Knowledge System",
  "entities": { ... },      // 实体类型和字段
  "relations": { ... },     // 类型化关系
  "lifecycle": { ... },     // 生命周期状态机
  "gates": { ... },         // 信任门禁
  "workflows": { ... },     // 工作流
  "artifacts": { ... }      // 哈希锚定产物
}
```

#### 创建自定义 profile：

```bash
# 从空白开始
llmwiki profile init

# 或使用模板
llmwiki template list        # 查看可用模板
llmwiki template inspect autosci  # 检查模板内容
llmwiki template init autosci     # 安装 AutoSci 研究模板

# 验证 profile
llmwiki profile validate

# 查看运行中的 profile
llmwiki profile show
```

### 2.3 AutoSci 研究模板实战

AutoSci 是一个完整的研究工作流系统：

```bash
mkdir research-project && cd research-project
llmwiki template init autosci
llmwiki profile validate
```

**AutoSci 定义的实体：**
- `paper` — 论文（字段：title, authors, doi, abstract, year, venue）
- `idea` — 研究想法（字段：hypothesis, status, priority）
- `experiment` — 实验（字段：method, dataset, results, reproducibility）
- `manuscript` — 手稿（字段：outline, draft, target_venue, status）

**AutoSci 定义的 Workflow：**
```bash
llmwiki workflow list
llmwiki workflow start literature-review
# 按工作流阶段推进
llmwiki workflow advance <run-id>
```

**Crossref 连接器：**
```bash
# 激活 Crossref 连接器
export LLMWIKI_CONNECTORS=crossref

# 在 .llmwiki/config.json 中配置
# { "connectors": { "crossref": { "contactEmail": "you@example.com" } } }

# 运行连接器导入文献
llmwiki connector list
llmwiki connector run crossref --input doi=10.1145/xxxxx
```

## 4. SDK 编程集成

### 4.1 TypeScript SDK 快速上手

```typescript
import { createWiki } from "llm-wiki-compiler";

const wiki = createWiki({ root: "/path/to/project" });

// 摄入源文件
const ingestResult = await wiki.ingest({ source: "https://example.com/doc.md" });

// 编译
const compileResult = await wiki.compile();
// → { compiled: 1, skipped: 0, deleted: 0, pages: ["my-concept"], errors: [] }

// 查询
const answer = await wiki.query({ question: "What is the key idea?" });

// 获取状态
const status = await wiki.status();
// → { pageCount: 5, sourceCount: 3, lastCompileTime: "..." }

// 导出
const exportResult = await wiki.export({ target: "json" });

// 运行评估
const evalReport = await wiki.eval({ suite: "fast" });
```

### 4.2 SDK 完整 API 参考

```typescript
interface Wiki {
  // 摄入
  ingest(input: IngestInput): Promise<IngestResult>;
  
  // 编译
  compile(options?: SdkCompileOptions): Promise<CompileResult>;
  
  // 查询
  query(input: { question: string; save?: boolean }): Promise<QueryResult>;
  
  // 上下文构建
  context(prompt: string, options?: ContextPackOptions): Promise<ContextPack>;
  
  // 读取
  status(): Promise<WikiStatus>;
  lint(): Promise<LintSummary>;
  eval(options?: { suite: "fast" | "full"; sampleSize?: number }): Promise<EvalReport>;
  
  // 导出
  export(options: { target: string; out?: string }): Promise<any>;
  
  // 页面操作
  readPage(slug: string): Promise<PageRecord | null>;
  listPages(): Promise<PageRef[]>;
  
  // 关系操作（CLP 必需）
  createRelation(input: AppendRelationInput): Promise<RelationRef>;
  
  // 生命周期操作（CLP 必需）
  transitionLifecycle(input: SdkTransitionLifecycleInput): Promise<void>;
  
  // 暂存操作
  stageEntityPage(input: SdkStageEntityPageInput): Promise<StagedChange>;
  promoteStagedPage(id: string): Promise<void>;
  
  // Artifact 操作
  writeArtifact(input: SdkWriteArtifactInput): Promise<ArtifactRef>;
  verifyArtifact(ref: ArtifactRef): Promise<ArtifactHealth>;
}
```

## 5. OKF（Open Knowledge Format）完整指南

### 5.1 设计哲学

OKF 是 Google Cloud 发起的开放格式倡议，llm-wiki-compiler 是 OKF 的生产者和消费者。其核心理念是：

> 编译后的知识应该可以像代码一样被**移植、审查、版本控制、共享**。

### 5.2 导出

```bash
# 导出为 OKF bundle
llmwiki export --target okf --out ./dist/okf

# OKF bundle 结构
dist/okf/
├── index.md           # OKF index
├── log.md             # Activity log
├── concepts/          # Compiled pages
├── queries/           # Saved answers
├── <entity-type>/     # Typed pages (if CLP is active)
└── profile.md         # Profile documentation (if CLP is active)
```

### 5.3 导入（审查优先）

```bash
# 预览导入
llmwiki import --okf ./dist/okf --dry-run

# 审查式导入（默认——进入审查队列）
llmwiki import --okf ./dist/okf

# 信任式导入（跳过审查——你信任 bundle 的内容和来源）
llmwiki import --okf ./dist/okf --trusted

# 导入后的页面进入审查队列
llmwiki review list
llmwiki review show <candidate-id>
llmwiki review approve <candidate-id>
```

### 5.4 跨工具知识共享

```
llmwiki export --target okf  →  OKF bundle
                                    ↓
                         Atomic Memory import
                         Obsidian vault
                         Other OKF-compatible tools
```

## 7. 高级集成模式

### 7.1 LangGraph + llmwiki → 自演进知识底座

参考我们的 ADN 调研报告，推荐架构：

```python
# LangGraph 中集成书记官 Agent
from langgraph.graph import StateGraph, END

# 定义 Scribe Node
def scribe_node(state):
    # 1. 从 Checkpointer 获取最新对话
    conversation = state["messages"][-10:]  # 最近 10 轮
    
    # 2. 构建 evidence pack（通过 MCP）
    context = mcp_call("get_context_pack", {
        "prompt": f"Assess knowledge novelty of: {conversation}"
    })
    
    # 3. 判断是否需要编译
    if context["has_novel_knowledge"]:
        # 4. 调用 MCP compile（或 stage for review）
        result = mcp_call("compile_wiki")
        return {"knowledge_updated": True, "compile_result": result}
    
    return {"knowledge_updated": False}

# 添加到图
graph = StateGraph(State)
graph.add_node("scribe", scribe_node)
graph.add_conditional_edges("main_agent", 
    lambda s: "scribe" if s.get("should_capture_knowledge") else END
)
```

### 7.2 llmwiki watch 模式

```bash
# 启动文件系统监听：sources/ 有变化时自动触发编译
llmwiki watch

# 带自定义并发
llmwiki watch --concurrency 8
```

内部实现使用 `chokidar`（fs.watch 的包装），在 `sources/` 目录有文件变更时自动运行增量编译。

### 7.3 CI/CD 质量门禁

```yaml
# 在 CI 中阻止低质量知识提交
- name: Wiki Quality Gate
  run: |
    llmwiki lint
    llmwiki eval --suite fast
    # 如果健康分低于某阈值，失败
    HEALTH=$(llmwiki eval --suite fast --json | jq '.healthScore')
    if [ "$HEALTH" -lt 80 ]; then
      echo "Wiki health score $HEALTH below threshold 80"
      exit 1
    fi
```

## 8. 进阶技巧

### 8.1 会话导入（`ingest-session`）

```bash
# 导入 Claude Code 会话
llmwiki ingest-session ~/claude.json

# 导入 Codex 会话
llmwiki ingest-session ~/codex-session.json

# 导入 Cursor 会话
llmwiki ingest-session ~/cursor-export.json
```

支持的适配器：`src/adapters/claude.ts`, `src/adapters/codex.ts`, `src/adapters/cursor.ts`

### 8.2 渐进式审查

```bash
# 配置审查策略（.llmwiki/config.json）
{
  "version": 1,
  "review": {
    "holdOnConfidence": "low",      // 低置信度页面自动扣留
    "holdOnContradiction": true,    // 有矛盾的页面自动扣留
    "holdOnNewSource": true,        // 新来源的页面自动扣留
    "maxAutoApprovePerCompile": 10  // 每次编译最多自动批准 10 页
  }
}
```

### 8.3 多语言输出

```bash
# 生成中文 Wiki
llmwiki compile --lang zh-CN

# 或通过环境变量
export LLMWIKI_OUTPUT_LANG=zh-CN
llmwiki compile

# 查询也用中文回答
llmwiki query "BGP 选路规则是什么？" --lang zh-CN
```
