# CHM 转文件夹 —— 完整流程文档

## 目标

将华为交换机命令参考 CHM 文件转换为带层级 README 导航的纯文本文件夹结构，方便大模型（LLM）逐层检索。

---

## 第一步：解包 CHM 文件

CHM 是微软 Compiled HTML Help 格式，本质是一个压缩归档。使用 7-Zip 解包：

```bash
# 安装 7-Zip（winget 在 Windows 上可用）
winget install 7zip.7zip

# 解包
7z.exe x "输入.chm" -o"输出目录" -y
```

解包后得到约 8951 个文件：
- `*.hhc` —— 目录树（Table of Contents），GB2312 编码
- `*.hhk` —— 关键字索引
- `*.html` —— 正文内容页，分布在 `v6r25c00/CLI/` 和 `toctopics/` 下
- `images/`, `public_sys-resources/` —— 图片和样式资源
- `#STRINGS`, `#TOPICS`, `#URLSTR` 等 —— CHM 内部索引文件

---

## 第二步：解析 .hhc 目录树

.hhc 文件是一个类 HTML 的站点地图，结构如下：

```html
<UL>
  <LI><OBJECT>
    <param name="Name" value="命令参考">
    <param name="Local" value="v6r25c00/CLI/dc_title.html">
  </OBJECT><UL>
    <LI><OBJECT>
      <param name="Name" value="基础配置">
      ...
    </OBJECT><UL>
      <LI><OBJECT>
        <param name="Name" value="alias">
        <param name="Local" value="v6r25c00/CLI/ALIASCONFIG(CLIOM).html">
      </OBJECT></LI>
    </UL></LI>
  </UL></LI>
</UL>
```

### 解析策略

1. 用正则匹配所有 `<OBJECT>...</OBJECT>` 块，提取 Name 和 Local
2. 统计每个 OBJECT 之前的 `<UL>` 和 `</UL>` 数量差，确定层级深度
3. 共解析出 8914 个条目，深度分布：`{1: 3, 2: 20, 3: 177, 4: 8714}`

### 关键点

- .hhc 文件编码是 GB2312，需用 `encoding='gb2312'` 读取
- `<UL>` 标签可能和 `<LI><OBJECT>` 在同一行，需要正确处理先后顺序

## 第三步：提取 HTML 纯文本

### 挑战

- HTML 文件也是 GB2312 编码
- Python 的 HTMLParser 不会自动处理自闭合标签（`<meta>`, `<link>`, `<br>`），导致 `skip_depth` 计数错误

### 解决方案

实现自定义 HTMLTextExtractor，在 `handle_starttag` 中对自闭合标签立即回退 skip_depth：

```python
SELF_CLOSING = {'meta', 'link', 'br', 'hr', 'img', 'input', 'base', 'area', 'col', 'embed', 'source', 'track', 'wbr'}

def handle_starttag(self, tag, attrs):
    if tag in self.skip_tags:
        self.skip_depth += 1
    # 自闭合标签立即回退
    if tag in self.SELF_CLOSING and tag in self.skip_tags:
        if self.skip_depth > 0:
            self.skip_depth -= 1
```

跳过标签：`script`, `style`, `head`, `meta`, `link`, `noscript`

## 第四步：重建文件夹结构

### 层级判定

遍历 8914 个条目，每个条目：
- `depth` 决定路径栈的深度
- 如果下一个条目的 depth 更深 → 当前条目是**文件夹节点**（有子项）
- 否则是**叶子节点**（具体命令）

### Windows 长路径问题

嵌套多层后路径超过 260 字符限制（`MAX_PATH`）。解决：使用 `\\?\` 前缀（Windows 扩展路径语法）+ 文件名截断到 40 字符。

```python
OUTPUT_DIR = Path(r"\\?\C:\Users\chenjinyu\Desktop\机器学习\tmp\命令参考_文件夹")
os.makedirs(out_str, exist_ok=True)  # 原生 API 支持 \\?\ 前缀
```

### 文件命名

清理不安全的文件名字符 `[\\/:*?"<>|]`，限制最长 40 字符。

## 第五步：编写 README.md 导航体系

### 层次结构（4 层）

| 层级 | 数量 | 内容 |
|------|------|------|
| 根目录 | 1 | 总体介绍、数据统计、导航方法、典型查找示例 |
| 命令参考/ | 1 | 20 个章节对比表（名称 + 说明 + 典型场景） |
| 章节级 | 20 | 子目录用途表 + 适用场景 + 常用命令快速查找 |
| 命令类级 | 177 | 类别说明 + 该类别下所有命令的一行功能简介 |

### 生成方式

- **命令类级**（177个）：先用 Python 脚本批量提取每个命令文件的"命令功能"段，按文件夹名称的关键词匹配生成类别描述，再逐一写入 README
- **章节级**（20个）：人工读取每个章节下所有子目录的 README，总结该章节覆盖的功能域，编写导航表
- **顶层 + 根**（2个）：综合所有章节信息编写

### 设计原则

- 每一层 README 都包含"适用场景"，让大模型能根据用户意图快速定位
- 中层以上必须有"快速查找"列表，覆盖最常见需求
- 不写"包含xxx命令"这类无信息量的描述
