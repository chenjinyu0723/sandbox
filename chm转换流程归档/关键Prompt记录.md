## Prompt 2：添加 README 导航

> 接着你为每个层级写一个README.md，里面写了命令或每个文件夹的简要介绍，方便大模型选择哪个文件夹进去，然后最终选择哪个命令。你从最低层级的（也就是有各个命令的文件夹）开始写，然后不断的往高层级写md，为大模型写导航md。你必须自己去读每个文件，你命令那一层的说明可以参照命令文本里的功能写，然后往上写的时候要总结当前层级每个子文件夹都是干嘛，再写下去成md（不许用python，必须你自己读，特别是中间层和顶层，你不能草草写个包含啥啥命令就了事，你要总结这个文件夹是干嘛的，后续做大模型搜索才方便）。你先把当前所有的README.md给删了再开始

**核心要求拆解**：
1. 从底层往上写（leaf → branch → root）
2. 底层参照命令的"命令功能"段来写简介
3. 中层和顶层必须**总结功能用途**，不能只列文件名
4. 目的是**为大模型导航**，要写得方便搜索
5. 必须自己读文件内容，不能只靠文件名猜测

**影响**：
- 177 个底层 README 通过提取每条命令的"命令功能"来写简介
- 20 个章节 README 都包含了"用途 + 适用场景 + 快速查找"
- 顶层 README 提供了多级导航路径
